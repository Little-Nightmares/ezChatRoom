## 二、安装开发环境

### 2.1 安装 Qt 6.11.0

**方法一：Qt 官方安装器（推荐）**

1. 打开 [Qt 官网下载页](https://www.qt.io/download-qt-installer)
2. 下载 Qt Online Installer（在线安装器）
3. 运行安装器，登录/注册 Qt 账号
4. 在组件选择界面，勾选：
   - **Qt 6.11.0** → 你的操作系统对应的编译器：
     - Windows: `MSVC 2022 64-bit` 或 `MinGW 11.2.0 64-bit`
     - macOS: `Clang`
     - Linux: `GCC`
   - **Qt 6.11.0** → Additional Libraries → 勾选 `Qt Network Auth`、`Qt SQL`
   - **Developer and Designer Tools** → 勾选 `CMake`、`Qt Creator`
5. 选择安装路径（建议默认），点击安装

**方法二：命令行安装（Linux）**

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install build-essential cmake qt6-base-dev qt6-declarative-dev \
                 qt6-network-dev qt6-sql-dev libssl-dev
```

### 2.2 安装 IDE（推荐 Qt Creator）

Qt Creator 是 Qt 官方 IDE，对 Qt 项目支持最好。

- 如果用 Qt 安装器安装，Qt Creator 已经包含在内
- 单独下载：[Qt Creator 下载](https://www.qt.io/product/development-tools)

**也可以用其他 IDE：**
- **VS Code**：安装 C/C++ 扩展 + CMake Tools 扩展
- **CLion**：JetBrains 出品，CMake 支持很好（学生可免费申请许可证）

### 2.3 验证安装

打开终端/命令行，输入：

```bash
cmake --version    # 应显示 3.16+
qmake --version    # 应显示 Qt 6.11.0
```

## 三、打开并编译项目

### 3.1 用 Qt Creator 打开（最简单）

1. 打开 Qt Creator
2. 点击 **文件 → 打开文件或项目**
3. 选择 `ChatRoom/CMakeLists.txt`
4. Qt Creator 会自动配置项目
5. 点击左下角的 **绿色三角形 ▶** 按钮编译运行

### 3.2 用命令行编译

```bash
cd ChatRoom
mkdir build && cd build
cmake .. -DCMAKE_PREFIX_PATH="C:/Qt/6.11.0/msvc2022_64"  # Windows 路径示例
cmake --build .
```

> **注意**：`CMAKE_PREFIX_PATH` 要改成你实际的 Qt 安装路径。

### 3.3 第一次编译预期结果

- 客户端编译成功后运行 → 会弹出一个空白窗口（标题"ChatRoom"），这是正常的
- 服务端编译成功后运行 → 命令行窗口一闪而过，这也是正常的（因为 main.cpp 里只有空骨架）

## 四、推荐开发顺序（从简单到复杂）

**核心原则：先让一个功能完整跑通，再开发下一个。**

```
阶段 1：基础框架搭建（1-2 周）
  └─ 让客户端能显示界面，服务端能启动监听

阶段 2：通信链路打通（2-3 周）
  └─ 客户端能连上服务端，能互发简单消息

阶段 3：用户系统（1-2 周）
  └─ 注册、登录、用户信息管理

阶段 4：好友系统（1-2 周）
  └─ 添加好友、好友列表、好友请求

阶段 5：聊天功能完善（1-2 周）
  └─ 聊天界面、消息记录、离线消息

阶段 6：加密功能（1 周）
  └─ AES + RSA 消息加密

阶段 7：优化与扩展（持续）
  └─ 文件传输、群聊、UI 美化等
```

## 五、每个阶段具体做什么（详细指引）

### 阶段 1：基础框架搭建

**目标**：客户端显示登录界面，服务端能启动。

#### 服务端（先做，因为客户端需要连接它）

**文件**：`server/src/main.cpp`、`server/src/core/TcpServer.h/.cpp`

1. **修改 `server/src/core/TcpServer.h`**：
   - `TcpServer` 继承自 `QTcpServer`
   - 添加一个 `start()` 槽函数，调用 `listen()` 监听端口

2. **修改 `server/src/core/TcpServer.cpp`**：
   ```cpp
   void TcpServer::start(quint16 port)
   {
       if (!this->listen(QHostAddress::Any, port)) {
           qDebug() << "Server start failed:" << this->errorString();
       } else {
           qDebug() << "Server started on port" << port;
       }
   }
   ```

3. **修改 `server/src/main.cpp`**：
   ```cpp
   #include <QCoreApplication>
   #include "core/TcpServer.h"

   int main(int argc, char *argv[])
   {
       QCoreApplication app(argc, argv);
       server::TcpServer server;
       server.start(6667);  // 监听 6667 端口
       return app.exec();
   }
   ```

4. **编译运行** → 终端应显示 `Server started on port 6667`

#### 客户端

**文件**：`client/qml/main.qml`、`client/qml/pages/LoginPage.qml`

1. **修改 `client/qml/pages/LoginPage.qml`**，做一个简单登录界面：
   ```qml
   import QtQuick
   import QtQuick.Controls
   import QtQuick.Layouts

   Page {
       title: qsTr("Login")
       ColumnLayout {
           anchors.centerIn: parent
           spacing: 10
           TextField {
               id: usernameField
               placeholderText: qsTr("用户名")
               Layout.preferredWidth: 300
           }
           TextField {
               id: passwordField
               placeholderText: qsTr("密码")
               echoMode: TextInput.Password
               Layout.preferredWidth: 300
           }
           Button {
               text: qsTr("登录")
               Layout.preferredWidth: 300
               onClicked: {
                   console.log("Login clicked:", usernameField.text)
               }
           }
       }
   }
   ```

2. **修改 `client/qml/main.qml`**，加载登录页面：
   ```qml
   import QtQuick
   import QtQuick.Controls

   ApplicationWindow {
       id: root
       width: 400
       height: 600
       visible: true
       title: qsTr("ChatRoom")

       LoginPage { anchors.fill: parent }
   }
   ```

3. **编译运行** → 应看到登录界面

---

### 阶段 2：通信链路打通

**目标**：客户端点击登录 → 连接服务端 → 服务端收到连接

#### 需要修改的文件

| 文件 | 做什么 |
|------|--------|
| `client/src/network/TcpClient.h/.cpp` | 封装 QTcpSocket，提供 connectToServer() 方法 |
| `server/src/core/TcpServer.h/.cpp` | 重写 incomingConnection()，处理新连接 |
| `server/src/core/ClientSession.h/.cpp` | 管理单个客户端连接 |
| `client/src/qmlbridge/UserController.h/.cpp` | 暴露连接逻辑给 QML |

#### 关键概念：Qt 信号与槽

这是 Qt 最核心的机制，**必须理解**：

```cpp
// 信号：当某件事发生时发出
signals:
    void connected();
    void dataReceived(QByteArray data);

// 槽：收到信号后执行的函数
public slots:
    void onConnected() {
        qDebug() << "Connected to server!";
    }

// 连接信号和槽
connect(socket, &QTcpSocket::connected, this, &MyClass::onConnected);
```

#### 开发步骤

1. 在 `TcpClient` 中创建 `QTcpSocket`，提供 `connectToServer(host, port)` 方法
2. 在 `TcpServer` 中重写 `incomingConnection()`，为每个新连接创建 `ClientSession`
3. 在 `UserController` 中调用 `TcpClient`，暴露给 QML
4. 在 QML 登录按钮的 `onClicked` 中调用 C++ 的连接方法

---

### 阶段 3-7：后续阶段

每个阶段的开发模式相同：

1. **确定要改哪些文件**（参考项目结构说明中的模块职责）
2. **先写 C++ 后端逻辑**（.h 定义接口，.cpp 实现功能）
3. **再写 QML 界面**（调用后端暴露的接口）
4. **编译测试**


## 八、常见问题

### Q1：编译报错 "Qt6 not found"
**解决**：CMake 找不到 Qt，需要指定路径：
```bash
cmake .. -DCMAKE_PREFIX_PATH="你的Qt安装路径/6.11.0/编译器版本"
```

### Q2：QML 文件修改后没效果
**解决**：需要重新编译（Ctrl+B），QML 资源文件（.qrc）打包后才会更新。

### Q3：信号和槽连接报错
**解决**：检查函数签名是否完全匹配，参数类型必须一致。

### Q4：中文乱码
**解决**：确保所有源文件保存为 **UTF-8** 编码。在 Qt Creator 中：
工具 → 选项 → 文本编辑器 → 行为 → 默认编码 → UTF-8

### Q5：链接错误 "undefined reference to vtable"
**解决**：通常是因为忘了在 .cpp 中实现 .h 中声明的函数，或者忘了 `#include` 对应的头文件。

## 十、给你的建议

1. **不要试图一次写完所有功能**，按阶段来，每完成一个阶段就测试
2. **先跑通再优化**，第一版代码丑一点没关系，能跑就行
3. **多看 Qt 官方文档和示例**，Qt 的文档质量非常高
4. **遇到 bug 先 qDebug**，90% 的问题靠打印日志就能定位
5. **善用 AI 工具**（比如我），遇到不懂的代码或报错直接问
6. **C++ 基础最重要**，如果 C++ 不熟，先把类、继承、指针搞懂再写 Qt
